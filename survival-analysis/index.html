基于 Kaplan-Meier 与 Cox 模型的客户流失预测

2026-04-27 / 预计阅读 15 分钟 / 生存分析 · 客户流失 · Kaplan-Meier · Cox 回归

1. 背景与目标

为什么用生存分析？

传统的客户流失预测方法（如逻辑回归、决策树）通常将流失视为二元分类问题，忽略了时间维度的重要信息。生存分析（Survival Analysis）是一组专门用于分析"事件发生时间"的统计方法，能够：

  • 处理删失数据（Censored Data）：客户在观察期结束时仍未流失
  • 估计生存函数：客户在不同时间点的留存概率
  • 识别风险因素：哪些因素影响客户流失速度

业务场景

电信行业面临激烈的客户竞争，获取新客户的成本远高于保留现有客户。通过生存分析，我们可以：

  1. 预测客户在不同时间点的流失风险
  2. 识别高风险客户群体，实施针对性干预
  3. 计算客户生命周期价值（CLV），优化获客策略

分析目标

  • 识别流失风险因素：哪些服务或客户特征影响留存
  • 预测客户价值：计算不同客户群体的 CLV
  • 提供业务建议：基于分析结果提出可落地的策略

3. Kaplan-Meier 生存分析

方法原理

Kaplan-Meier 是一种非参数方法，用于估计生存函数 S(t)，即个体在时间 t 仍然存活的概率。

核心公式：

S(t) = ∏ti ≤ t (1 - di/ni)

其中 di 是在时间 ti 发生事件的数量，ni 是在时间 ti 面临风险的个体数量。

整体生存曲线

图 2：Kaplan-Meier 整体生存曲线（含 95% 置信区间）

关键发现

中位生存时间：20 个月

这意味着有 50% 的月付互联网客户会在 20 个月内流失。置信区间随时间变宽，因为留存时间越长的客户数量越少，不确定性增加。

生存概率变化趋势

0 月100%所有客户初始留存
10 月~70%约 30% 的客户在前 10 个月内流失
20 月50%中位数，半数客户已流失

48 月~24%约四分之一客户长期留存

4. Cox 比例风险模型

方法原理

Cox 比例风险模型是一种半参数方法，支持多变量分析，可以同时考虑多个协变量对生存时间的影响。

模型方程：

h(t|X) = h0(t) × exp(β1X1 + β2X2 + ... + βpXp)

其中 h(t|X) 是条件风险函数，h0(t) 是基线风险，βi 是回归系数。

风险比（Hazard Ratio）解读

  • HR > 1：增加风险（加速流失）
  • HR < 1：降低风险（延缓流失）
  • HR = 1：无影响

模型变量

基于 Kaplan-Meier 分析结果，选择以下显著变量进入 Cox 模型：

  • Dependents_Yes：是否有受抚养人
  • InternetService_DSL：是否使用 DSL 服务
  • OnlineBackup_Yes：是否有在线备份服务
  • TechSupport_Yes：是否有技术支持服务

模型结果

结果解读

通俗解读

  • 在线备份服务：有在线备份的客户流失风险降低 54%（HR=0.46）
  • 技术支持服务：有技术支持的客户流失风险降低 47%（HR=0.53）
  • 有受抚养人：有受抚养人的客户流失风险降低 28%（HR=0.72）
  • DSL 服务：DSL 客户比光纤客户流失风险降低 20%（HR=0.80）

关键发现

在线备份服务是最强的保护因素（HR=0.46），其次是技术支持服务（HR=0.53）。所有纳入模型的变量都是保护因素（HR < 1），且统计显著（p < 0.005）。

比例风险假设检验

Cox 模型假设各组之间的风险比随时间保持恒定。我们使用 Schoenfeld 残差检验和 Log-log Kaplan-Meier 图进行验证。

图 7：Log-log Kaplan-Meier 图（用于检验比例风险假设）

如果 Log-log 图中的曲线平行，则满足比例风险假设。如果曲线交叉或不平行，可能需要考虑分层模型或时间依赖变量。

5. AFT 模型对比

方法原理

加速失效时间（Accelerated Failure Time, AFT）模型是一种参数方法，假设生存时间服从特定分布。与 Cox 模型不同，AFT 模型直接估计生存时间的变化。

模型方程：

SA(t) = SB(λt)

其中 λ 是加速因子：λ > 1 表示 B 组生存时间被"延长"，λ < 1 表示被"加速"。

Cox vs AFT 对比

特性 Cox 模型 AFT 模型
类型 半参数 参数
估计量 风险比（Hazard Ratio） 时间加速因子
分布假设 无 需要指定分布（Log-Logistic）
可解释性 风险变化 时间变化（更直观）
Concordance 0.64 0.73

Log-Logistic AFT 模型结果

图 8：AFT 模型时间加速因子

变量 exp(coef) 解读
OnlineSecurity_Yes 2.37 有网络安全服务的客户留存时间延长 137%
OnlineBackup_Yes 2.25 有在线备份的客户留存时间延长 125%
PaymentMethod_Credit card 2.22 信用卡自动付款客户留存时间延长 122%
TechSupport_Yes 1.99 有技术支持的客户留存时间延长 99%
Partner_Yes 1.97 有伴侣的客户留存时间延长 97%

结论

AFT 模型在本数据集上表现更好（Concordance 0.73 vs 0.64），且时间加速因子的解释更直观，便于业务理解。

6. 客户生命周期价值 (CLV)

计算方法

CLV 计算公式：

CLV = Σt=0T [S(t) × M] / (1+r)t

其中 S(t) 是时间 t 的生存概率，M 是月利润，r 是月折现率，T 是计算周期。

参数设置

  • 月利润：$30
  • 年折现率：10%
  • 计算周期：36 个月

  2. 低风险客户价值最高，应优先获取和保留
  3. 应重点推广在线备份和技术支持服务，提升客户价值

7. 结论与业务建议

核心发现

1

最强保护因素

在线备份服务（HR=0.46）和技术支持服务（HR=0.53）是客户留存的最强保护因素。有这些服务的客户流失风险降低近一半。

2

非显著因素

性别不是客户流失的显著影响因素（p > 0.05），营销策略无需区分性别。

3

模型对比

AFT 模型（Concordance=0.73）优于 Cox 模型（Concordance=0.64），且时间加速因子的解释更直观。

4

客户价值差异

低风险客户 36 个月 CLV 比高风险客户高 43%，所有客户类型都能在 4 个月内收回获客成本。

业务建议

客户保留策略

  • 重点推广在线备份和技术支持服务
  • 鼓励客户使用自动付款方式
  • 关注光纤服务客户的流失风险，提升服务质量

客户获取策略

  • 优先获取有受抚养人、有伴侣的客户群体
  • 推广 DSL 服务套餐，降低光纤客户流失风险
  • 设计包含增值服务的组合套餐

服务改进方向

  • 提升光纤服务的客户体验
  • 开发更多增值服务提高客户粘性
  • 建立客户流失预警系统

方法局限性

  • 数据局限：数据为样本数据，实际应用需结合真实业务数据
  • 模型假设：Cox 模型需满足比例风险假设，AFT 模型需指定分布
  • 外部因素：未考虑竞争对手、市场环境等外部因素
  • 时间范围：CLV 计算基于 36 个月预测，长期预测不确定性较高

8. 技术细节

# 数据预处理
import pandas as pd

# 加载数据
df = pd.read_csv("Telco-Customer-Churn.csv")

# 将 Churn 列转换为数值: Yes -> 1, No -> 0
df['churn'] = df['Churn'].map({'Yes': 1, 'No': 0})

# 筛选条件：
# 1. 合同类型为月付 (Month-to-month)
# 2. 有互联网服务 (InternetService != 'No')
telco_pd = df[
    (df['Contract'] == 'Month-to-month') &
    (df['InternetService'] != 'No')
].copy()

print(f"筛选后数据集大小: {telco_pd.shape[0]} 行")
print(f"流失比例: {telco_pd['churn'].mean()*100:.2f}%")
from lifelines import KaplanMeierFitter

# 创建 Kaplan-Meier 拟合器
kmf = KaplanMeierFitter()

# 准备数据
T = telco_pd['tenure']  # 持续时间
C = telco_pd['churn'].astype(float)  # 事件观察

# 拟合模型
kmf.fit(T, C)

print(f"中位生存时间: {kmf.median_survival_time_} 个月")

# 绘制生存曲线
import matplotlib.pyplot as plt
plt.figure(figsize=(12, 6))
kmf.plot(title='Kaplan-Meier 生存曲线',
         xlabel='时间（月）',
         ylabel='生存概率')
plt.show()
from lifelines import CoxPHFitter
import pandas as pd

# One-Hot 编码
encode_cols = ['Dependents', 'InternetService', 'OnlineBackup', 'TechSupport']
encoded_pd = pd.get_dummies(telco_pd, columns=encode_cols, prefix=encode_cols)

# 选择建模变量
survival_pd = encoded_pd[['churn', 'tenure',
                          'Dependents_Yes',
                          'InternetService_DSL',
                          'OnlineBackup_Yes',
                          'TechSupport_Yes']]

# 拟合 Cox 模型
cph = CoxPHFitter(alpha=0.05)
cph.fit(survival_pd, duration_col='tenure', event_col='churn')

# 打印模型摘要
cph.print_summary()

# 检验比例风险假设
cph.check_assumptions(survival_pd, p_value_threshold=0.05)
def calculate_clv(customer_features, monthly_profit=30, irr=0.10, months=36):
    """
    计算客户生命周期价值

    参数:
        customer_features: dict, 客户特征
        monthly_profit: float, 月利润
        irr: float, 年折现率
        months: int, 计算月数
    """
    import pandas as pd

    # 将客户特征转换为 DataFrame
    customer_df = pd.DataFrame([customer_features])

    # 预测生存概率
    survival_probs = cph.predict_survival_function(customer_df)

    # 创建结果表
    result = pd.DataFrame()
    result['月份'] = range(0, months + 1)
    result['生存概率'] = survival_probs.values[:months+1, 0]
    result['预期月利润'] = result['生存概率'] * monthly_profit

    # 计算 NPV
    monthly_irr = irr / 12
    result['NPV'] = result['预期月利润'] / ((1 + monthly_irr) ** result['月份'])
    result['累计NPV'] = result['NPV'].cumsum()

    return result

# 示例：计算低风险客户的 CLV
low_risk_customer = {
    'Dependents_Yes': 1,
    'InternetService_DSL': 1,
    'OnlineBackup_Yes': 1,
    'TechSupport_Yes': 1
}

clv_result = calculate_clv(low_risk_customer)
print(f"36 个月 CLV: ${clv_result.loc[36, '累计NPV']:.2f}")
